## Spacing Scale

8-point scale. All layout spacing resolves from these tokens (Tailwind v4 default 4px-base scale aligns; declared subset below is the sanctioned vocabulary for this phase).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, inline padding |
| sm | 8px | Compact element spacing, pill padding-y |
| md | 16px | Default element spacing, button padding-x |
| lg | 24px | Inter-element rhythm inside hero, nav padding |
| xl | 32px | Mobile section vertical padding, CTA group gap |
| 2xl | 48px | Hero internal block separation |
| 3xl | 64px | Desktop section vertical padding / major breaks |

**Exceptions:**
- Interactive touch targets have a **minimum hit area of 44×44px** even when the visual element is smaller (mobile usability at 360px — LNCH-01). Achieve via padding, not by enlarging the visual.
- Hero vertical centering uses `min-h-[100svh]` (small-viewport height) rather than a spacing token, to avoid mobile URL-bar jump.

**Radius scale** (part of the single token source — DSYS-01):

| Token | Value | Usage |
|-------|-------|-------|
| radius-sm | 6px | Tech pills, small chips |
| radius-md | 10px | Buttons |
| radius-lg | 16px | Cards |
| radius-pill | 9999px | Fully-rounded pills / icon buttons |

## Typography

Two text weights total across the system: **400** and **600** (DSYS keeps the weight vocabulary tight). Display and mono use their family at one of these same two weights — no third weight is loaded.

| Role | Font | Size | Weight | Line Height | Notes |
|------|------|------|--------|-------------|-------|
| Display (hero LCP h1) | Clash Display | `clamp(2.75rem, 6vw, 4.5rem)` (~44→72px) | 600 | 1.05 | The **only** gradient-filled text. Renders at full opacity, never animated (HERO-03). Tracking `-0.02em`. |
| Heading (section h2) | Clash Display | 28px (`1.75rem`) | 600 | 1.2 | Solid near-white, NOT gradient. |
| Body | Inter | 16px (`1rem`) | 400 | 1.5 | Solid near-white / muted. Sub-headline may scale to 18px on desktop. |
| Label / Kicker / Pill | JetBrains Mono | 14px (`0.875rem`) | 400 | 1.4 | Uppercase, tracking `0.08em`. Used for the "Computer Engineering @ UIUC" eyebrow, tech pills, nav. |
| Emphasis (button label / inline strong) | Inter | 16px | 600 | 1.5 | The semibold weight. |

- Body line-height **1.5**, heading/display line-height **1.05–1.2**.
- Clash Display Bold (700) is available but **reserved** — not used in this phase; keep the contract at one display weight (600).

## Color

Dark-first 60/30/10. Body text is **always solid near-white** — the gradient touches large display text only (DSYS-04, HERO-03, LNCH-03). All values target WCAG AA on the dark surface.

| Role | Value | Usage | Source |
|------|-------|-------|--------|
| Dominant (60%) | `#0A0A0F` | Page background / base surface | `[default]` |
| Secondary (30%) | `#14141F` | Elevated surfaces: cards, nav bar, pill backgrounds | `[default]` |
| Border / hairline | `#262633` | 1px borders, dividers, pill outlines | `[default]` |
| Text — primary | `#EDEDF2` | Body, headings, hero sub-headline (solid, ≥13:1 on dominant) | `[default]` |
| Text — muted | `#A1A1B5` | Secondary/meta text, kicker (≈6:1 on dominant — AA pass) | `[default]` |
| Accent — gradient stops | violet `#7C3AED` → blue `#3B82F6` → cyan `#22D3EE` | The signature violet→blue→cyan gradient | `[locked direction, default stops]` |
| Accent — solid (focus / single-hue) | `#3B82F6` (blue mid-stop) | Keyboard focus ring, single-color accent where a gradient is inappropriate | `[default]` |
| Destructive | not applicable this phase | No destructive actions exist in Phase 1 | — |

**Accent reserved for (explicit list — never "all interactive elements"):**
1. The hero **display h1** gradient text-fill (the one gradient headline).
2. **Primary CTA** button fill ("View Work") — gradient background.
3. **Keyboard focus ring** — solid `#3B82F6`, 2px, 2px offset (single hue, not gradient).
4. A single **ambient background glow** behind the hero (radial, low opacity ≤ 0.18, decorative only).
5. The **active/hover indicator** on nav links and the kicker underline accent.

Everything else (secondary/tertiary CTAs, surfaces, body text, borders) uses the neutral dark palette — no accent. Gradient on display headings must keep the lightest stop bright enough that the largest text still clears 3:1 (large-text AA).

## Motion & Interaction Contract (HERO-03, DSYS-04)

Single animation library: **Motion** (`motion/react`), client components only (`"use client"`). Wrap the app in `<MotionConfig reducedMotion="user">` so every animation auto-disables under `prefers-reduced-motion`.

| Element | Motion | Rule |
|---------|--------|------|
| Hero display **h1 (LCP)** | **NONE** | Must render immediately at full opacity. No opacity-fade, no transform, no `whileInView` on the LCP element. Hard requirement (HERO-03). |
| Hero kicker, sub-headline, CTA group | Subtle entrance allowed | Fade + translate ≤ 8px, ≤ 400ms, slight stagger. Never gate the LCP h1 behind these. Fully disabled under reduced-motion. |
| Ambient background gradient glow | Subtle drift only | Slow (8–12s) low-amplitude background-position/opacity loop. Decorative, non-essential. **Stops entirely** under `prefers-reduced-motion`. |
| Buttons / nav links (hover) | CSS transition | ≤ 150ms color/opacity/translate. No layout shift. |
| Scroll reveals (other sections) | Deferred | Full scroll-reveal layer is Phase 4 (`LazyMotion`); Phase 1 keeps motion minimal and hero-local. |

**Focus & keyboard (DSYS-04):** Every interactive element shows a visible focus ring (`#3B82F6`, 2px, 2px offset). Never remove `outline` without an equivalent replacement. Tab order follows visual order. Skip-to-content link recommended.

**Reduced motion:** `MotionConfig reducedMotion="user"` plus a CSS `@media (prefers-reduced-motion: reduce)` guard for the ambient glow keyframes.

## Fonts, Assets & basePath (DSYS-02, LNCH-05)

- **No FOUT/FOIT/CLS:** all three fonts self-hosted via `next/font` (`local` for Clash Display, `google` for Inter + JetBrains Mono). `display: "swap"` with the variables wired into `@theme inline`. Self-hosting means zero third-party font request.
- **basePath:** static export served from a project page → `basePath` + `assetPrefix` = `/jackyzhang`. `next/font` handles its own paths; any `public/` asset (e.g. `resume.pdf`, static OG, hero raster) must be referenced basePath-aware. Verify no `href="#"` placeholders and no broken asset paths on the deployed URL.
- **Images:** `unoptimized` (static export). Pre-size/compress source assets; explicit `width`/`height` to reserve space (no CLS).
