## Phase Goal

**As a** recruiter or hiring manager, **I want to** land on a fast, on-brand hero that names Jacky, his identity, and his builder arc with clear next-step CTAs, **so that** within seconds I can tell he is a serious builder worth interviewing.

> **This plan:** build the reusable, content-agnostic UI primitives (DSYS-03). Hero + Nav (01-04) and every later phase compose FROM these — this is the integration contract. Primitives must exist before the hero composes them (Walking-Skeleton sequencing step 3).

<objective>
Build the six-primitive design-system library — Button, Card, TechPill, Section, Timeline (shell) — as content-agnostic components styled only from the `@theme` tokens, with the DSYS-04 focus ring baked into every interactive surface.

Purpose: DSYS-03 requires reusable primitives that the hero/nav and Phases 2–3 compose rather than re-implement. Building them as a consistent set now (even Timeline, which is populated in Phase 3) locks the integration contract and prevents bespoke one-offs.
Output: Five primitive component files under `src/components/ui/`, each importing `cn`, styling from tokens, RSC-safe.
</objective>
