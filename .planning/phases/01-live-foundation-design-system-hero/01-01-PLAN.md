</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| npm registry → local/CI install | Third-party package code enters the build (supply chain) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-SC | Tampering | npm installs (`motion`, `lucide-react`, `clsx`, `tailwind-merge`, scaffold deps) | mitigate | All packages are canonical top-tier with explicit "Approved" verdicts in RESEARCH §"Package Legitimacy Audit" (no [ASSUMED]/[SUS]/[SLOP]); pin via `package-lock.json` + `npm ci` in CI. No blocking legitimacy checkpoint required. |
| T-01-01 | Elevation of Privilege | GitHub Actions workflow permissions | mitigate | Workflow uses least-privilege `permissions: { contents: read, pages: write, id-token: write }` only; no broad write scopes. |
| T-01-02 | Info disclosure | CI build env | accept | Only `NEXT_PUBLIC_BASE_PATH` (a public, non-secret value) is injected; no secrets exist for a static no-auth site. |
</threat_model>
